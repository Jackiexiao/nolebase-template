## String 和 StringBuilder的区别

```txt
String:
	1. JVM创建了一个String对象，内容是 "你好"，让 s 指向它。
	2. 当执行 + "，世界！" 时，JVM并没有修改原来的"你好"石板。
	3. 它创建了一个全新的String对象，内容是 "你好，世界！"。
	4. 然后，它让变量 s 不再指向旧的"你好"石板，而是指向这块全新的"你好，世界！"石板。
	5. 那块旧的"你好"石板，就成了没人要的垃圾，等着被垃圾回收器（GC）清理掉。
	结论：String 是不可变的。任何对 String 的修改操作，都会导致创建新的 String 对象，这在大量、频繁修改字符串的场景下，会非常浪费内存和CPU
StringBuilder:
	6. 创建了一个 StringBuilder 对象，它内部有个字符数组（白板），写着"你好"。
	7. 当执行 append() 方法时，它直接在内部的字符数组后面加上了 "，世界！"。
	8. 自始至终，都只有一个 StringBuilder 对象。没有创建任何新对象。
	结论：StringBuilder 是可变的。它被设计出来，就是为了高效地进行字符串的拼接和修改操作。
```
