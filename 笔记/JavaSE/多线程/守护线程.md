# 🧵 什么是守护线程？

**守护线程（Daemon Thread）**是一种在后台默默运行、为其他线程提供服务的线程。  
它最大的特点是：

> **当所有非守护线程（用户线程）结束后，JVM 会自动退出，不会等待守护线程执行完。**

也就是说：  
守护线程 **不会阻止 JVM 退出**。

<!-- prettier-ignore -->
> [!NOTE] 
> 	当其它的非守护线程执行完毕之后，守护线程会陆续结束
> 	通俗易懂：当女神结束后，备胎也没有存在的必要了

```Java
public class MultiDemo06 {
    public static void main(String[] args) {
        MyTread03 t1 = new MyTread03();
        MyThread04 t2 = new MyThread04();
        t1.setName("线程1");
        t2.setName("线程2");
        //将第二个线程设置为守护线程
        t2.setDaemon(true);
        t1.start();
        t2.start();
    }
}
```
|类型|是否阻止 JVM 退出|适用场景|
|---|---|---|
|**用户线程 (User Thread)**|✅ 会阻止|主业务逻辑、主要流程|
|**守护线程 (Daemon Thread)**|❌ 不会阻止|日志、监控、GC、心跳、后台定时任务|
---
典型例子：  
**垃圾回收器 GC** 就是守护线程。
